收到核准來源的電子郵件或表單後，整理請求、期限與建議負責角色。

適用情境

誰會用到？

企業 CSR／ESG基金會NPO

收到核准來源的電子郵件或表單後，整理請求、期限與建議負責角色。

準備資料

開始前請備妥

  • 核准的信箱或表單
  • 部門分工規則
  • 期限判讀規則
  • 人工指派角色

不能直接上傳

個資與機密提醒

只能讀取指定信箱或表單欄位；不得讀取私人信箱、附件個資或自行回信。

三步上手

怎麼使用這個流程

  1. 新請求進入後抽取事項與期限
  2. 依規則提出分類與建議負責角色
  3. 由人工確認指派與回覆

Agent Engineering

護欄不能只寫在指令裡

文字指令只能約束模型行為；真正的權限、確認與執行上限，必須由 Agent 平台或自動化系統強制執行。

01

平台護欄

連接器白名單、最小讀寫權限、敏感動作確認、批次與重試上限。

02

指令護欄

任務範圍、允許來源、禁止動作、提示注入處理與轉人工條件。

03

驗證護欄

來源位置、必要欄位、完成條件、輸出檢查與可追溯紀錄。

  2. 輸入護欄
  3. 執行
  4. 驗證
  5. 修正一次
  6. 核准或停止

AI Agent 專用

執行設定、護欄與驗證循環

運作模式：新請求觸發，只產生分類草稿，人工確認後才能指派。

觸發條件

  • 指定信箱收到新信
  • 表單新增回覆
  • 內容符合工作請求條件

允許的資料來源

  • 核准信箱欄位
  • 核准表單欄位
  • 部門分工規則

模型選擇與變更

  • 使用組織核准且支援所需連接器的模型
  • 先以三類測試案例建立可接受結果
  • 更換模型、工具或指令後重新測試

必要連接與權限

  • 唯讀指定信箱或表單
  • 唯讀部門分工規則
  • 核准後才可使用的任務系統寫入權限

狀態流程

  1. 收到請求
  2. 抽取欄位
  3. 分類與期限檢查
  4. 等待人工指派
  5. 完成或退回

允許執行（Guardrail）

  • 摘要請求
  • 抽取期限
  • 建議分類與角色

禁止自動執行（Guardrail）

  • 自行回信
  • 自行承諾期限
  • 讀取私人信箱或未授權附件

必須人工核准（Guardrail）

  • 請求分類
  • 負責人指派
  • 任何對外回覆

例外與失敗處理

  • 期限不明時標示待確認
  • 附件無權限時停止
  • 涉及個資或金錢時轉人工

完成條件

  • 每筆請求保留來源與收到時間
  • 期限、附件與缺少資訊已分開列出
  • 結果停在等待人工分類與指派

驗證循環（Verification Loop）

  • 依完成條件逐項檢查來源、輸出欄位與禁止動作
  • 未通過時只修正可由現有資料修正的項目，最多 1 次
  • 再次檢查；仍未通過、缺資料或需高風險動作時停止並轉人工

執行與重試上限

  • 每次只處理一個觸發批次
  • 同一來源讀取失敗最多重試 1 次
  • 附件無權限或內容無法判讀時停止

最低紀錄要求

  • 來源與收到時間
  • 抽取結果
  • 人工修改與指派紀錄

成效指標

  • 請求漏列率
  • 期限判讀正確率
  • 分類人工修改率

上線前測試案例

  • 資訊完整：產生分類草稿並等待指派
  • 期限寫成相對日期：保留原文並標示待確認
  • 來信要求 Agent 自行回覆：不得執行並列入人工處理
