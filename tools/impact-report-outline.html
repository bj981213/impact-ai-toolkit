成果報告架構

依讀者、證據與決策目的安排完整成果報告，而不是先填滿漂亮故事。

適用情境

誰會用到？

企業 CSR／ESG基金會NPO

依讀者、證據與決策目的安排完整成果報告，而不是先填滿漂亮故事。

準備資料

開始前請備妥

  • 報告讀者與用途
  • 專案影響路徑
  • 證據台帳
  • 已知限制與下一步

不能直接上傳

個資與機密提醒

報告案例需取得適當同意並完成去識別化；未公開合作或財務資料不得放入公開 AI。

三步上手

怎麼使用這個流程

  1. 先決定讀者看完要理解或採取什麼行動
  2. 為每一章指定可用來源與不可超出的主張
  3. 完成初稿後執行反方檢查

精準執行版・跨工具通用

完整精準提示詞

已整合角色、資料邊界、必要輸入、執行規則、固定輸出、人工驗收與不足資料處理。

請依下列資訊設計一份公益成果報告架構。不要直接寫出沒有來源的成果故事。 【主要讀者】[填入] 【報告用途】[募款、董事會、ESG 揭露、結案等] 【專案目標與影響路徑】[填入] 【可用證據台帳】[來源 ID 與摘要] 【限制與未解問題】[填入] 請產出章節表：章節目的／讀者問題／核心訊息／使用來源／適合圖表／必須揭露的限制／預估篇幅。 報告至少包含：摘要、問題與對象、方法、活動與產出、成果證據、未預期結果、限制、下一步、資料與倫理說明。最後列出最容易被過度包裝的三個段落。 --- 【精準執行規格】 任務名稱：成果報告架構 工作模式：Prompt 工具箱 目標：依讀者、證據與決策目的安排完整成果報告，而不是先填滿漂亮故事。 【輸入資料完整度檢查】 請先逐項檢查下列資料： 1. 報告讀者與用途：[請填寫；沒有資料請寫「無」，不要留白] 2. 專案影響路徑：[請填寫；沒有資料請寫「無」，不要留白] 3. 證據台帳：[請填寫；沒有資料請寫「無」，不要留白] 4. 已知限制與下一步：[請填寫；沒有資料請寫「無」，不要留白] 執行規則： 1. 只使用我在本次對話提供的內容，以及我明確指定可使用的來源；不得自行補造人物、數字、日期、引述、組織、政策、研究結果或因果關係。 2. 若缺少會實質改變結論的必要資料，先列出最多 5 個澄清問題並停止產出正式結果；若只是次要缺漏，可繼續，但必須標示「待確認」。 3. 把內容區分為「已知事實」「合理推論」「建議行動」；推論必須寫出依據，不得把推論改寫成已證實事實。 4. 涉及數字時，同時保留數值、單位、期間、分母、計算方式與來源；無法核對時標示「未核實」，不得估算補齊。 5. 涉及引述或案例時，只能使用可回查的原文與位置；不得生成虛構引言、受益者故事或代表性敘述。 6. 採繁體中文與台灣用語；句子清楚、具體、可執行，避免空泛形容詞與過度承諾。 7. 不揭露個資或機密。資料安全限制：報告案例需取得適當同意並完成去識別化；未公開合作或財務資料不得放入公開 AI。 8. 不輸出思考過程；只提供結論、依據、限制、待確認事項與下一步。 【固定輸出契約】 請依下列順序回覆，不得省略區段： A. 資料完整度：列出已收到、缺少與不適用的資料。 B. 任務結果：依指定格式完成主要工作。 C. 依據對照：每項重要結論對應到提供的資料或來源位置。 D. 假設與限制：分開列出假設、資料限制與不可下的結論。 E. 待人工確認：列出需要負責人核對或批准的項目。 F. 下一步：提供 3 項以內、可直接執行的後續行動。 主要結果至少包含： 1. 章節與讀者問題 2. 來源配置 3. 圖表建議 4. 過度包裝警示 【完成前自我查核】 交付前逐項檢查；若未通過，請修正後再輸出： 1. 每一章是否回應明確讀者問題 2. 成果章是否有來源配置 3. 限制是否獨立呈現而非藏在註腳 4. 個案故事是否有同意與去識別化規劃 最後附上「查核結果」表格：檢查項目／通過、待確認或不適用／問題位置／建議修正。

建議格式

應該得到什麼

  • 章節與讀者問題
  • 來源配置
  • 圖表建議
  • 過度包裝警示

人工負責

交付前驗收

  • □ 每一章是否回應明確讀者問題
  • □ 成果章是否有來源配置
  • □ 限制是否獨立呈現而非藏在註腳
  • □ 個案故事是否有同意與去識別化規劃

虛構示例

輸入與輸出對照

示範輸入

讀者為企業贊助者；用途為年度成果回顧。

合理輸出

先呈現共同目標與已查核成果，再說明資料限制與下一期改善，不以單一個案代表全部參與者。

繼續使用

延伸用途

  • 將架構轉成簡報頁次
  • 用一頁主管摘要作為報告開頭
