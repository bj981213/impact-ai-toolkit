NotebookLM 知識庫建置清單

規劃來源範圍、檔名、版本與更新節奏，建立能回查來源的專案知識庫。

適用情境

誰會用到？

企業 CSR／ESG基金會NPO

規劃來源範圍、檔名、版本與更新節奏，建立能回查來源的專案知識庫。

準備資料

開始前請備妥

  • 知識庫要回答的問題
  • 候選檔案與公開來源清單
  • 資料版本與權限
  • 維護責任角色

不能直接上傳

個資與機密提醒

上傳前確認帳號與分享權限；未去識別化個資、機密合約與未公開財務資料不得放入未核准空間。

三步上手

怎麼使用這個流程

  1. 先定義知識庫範圍與要回答的問題
  2. 盤點來源、版本、權限與可引用性
  3. 小量匯入後測試引用，再建立固定更新流程

AI Agent 專用

執行設定與治理規格

運作模式：人工啟動或每月排程的來源盤點與驗收規劃；網站只產生設定，不代替使用者上傳檔案。

觸發條件

  • 建立新知識庫或新增一批核准來源
  • 每月更新日到期
  • 來源版本、權限或分享範圍變更

允許的資料來源

  • 來源清單與中繼資料，不含機密正文
  • 文件擁有者、日期、版本與權限狀態
  • 知識庫目的、允許回答與禁止回答的問題

狀態流程

  1. 來源候選
  2. 權限與敏感度檢查
  3. 版本與重複檢查
  4. 納入、排除或待確認分類
  5. 驗收題產生
  6. 等待資料負責人批准
  7. 更新完成或退回處理

允許執行

  • 整理來源清單與版本關係
  • 標示過期、重複、衝突或缺少權限的來源
  • 產生可回答、不可回答與跨來源驗收題
  • 提出更新與權限複查清單

禁止自動執行

  • 自動上傳、刪除或分享文件
  • 把未核准個資、合約或財務資料列為可上傳
  • 找不到來源時以一般知識補寫專案事實

必須人工批准

  • 每份來源的納入與權限
  • 分享對象與可回答範圍
  • 過期來源的封存或移除

例外與失敗處理

  • 來源日期或版本不明時列為待確認
  • 兩份來源內容衝突時並列差異並停止定論
  • 權限不足或敏感度不明時排除於匯入清單外

最低紀錄要求

  • 每份來源的擁有者、版本、日期、權限與處置
  • 每次更新新增、移除與保留原因
  • 驗收題、引用結果與失敗案例

成效指標

  • 納入來源中日期、版本與擁有者完整率 100%
  • 驗收回答附可回查來源的比例
  • 過期或衝突來源被正確標示的比例
  • 權限複查準時完成率

跨工具通用

Agent 設定與系統提示詞

先替換所有以 [ ] 標示的欄位，再貼入 Agent 的自訂指令或系統設定。沒有設定區時，貼到新對話的第一則訊息；先用虛構資料測試。

規劃公益專案的 NotebookLM 知識庫。只設計清單與測試問題，不要要求我上傳敏感內容。 【知識庫用途】[填入] 【希望回答的問題】[填入] 【候選來源】[檔名、日期、版本、來源單位、權限] 【維護角色與更新頻率】[填入] 請產出： 1. 建議納入、排除、待確認的來源清單與理由。 2. 統一檔名、版本與日期規則。 3. 來源衝突與過期資料的處理方式。 4. 十題驗收問題，包含可回答、不可回答與跨來源比對。 5. 每月更新與權限複查清單。 所有回答都應要求引用來源；找不到時要回答「目前來源不足」，不得靠一般常識補齊專案事實。 【執行限制】 1. 所有以 [ ] 標示的欄位都必須替換。缺少會影響結論的資料時，只列出缺少欄位與補充方式，停止執行。 2. 只使用本次提供的資料與指定來源。不得補造人物、組織、數字、日期、引述、研究、政策或因果關係。 3. 事實、推論與建議分開標示。推論須附依據。 4. 數字保留單位、期間、分母、算法與來源；無法核對時標示「未核實」。 5. 使用繁體中文與台灣用語。刪除重複句、空泛形容詞及沒有依據的承諾。 6. 上傳前確認帳號與分享權限；未去識別化個資、機密合約與未公開財務資料不得放入未核准空間。 【交付前檢查】 - 知識庫範圍是否清楚 - 每份來源是否有日期、版本與權限 - 測試是否包含答不出來的問題 - 分享設定是否經資料負責人確認 未通過的項目先修正；無法修正時，標出問題位置與所缺資料。 【Agent 執行限制】 允許執行： - 整理來源清單與版本關係 - 標示過期、重複、衝突或缺少權限的來源 - 產生可回答、不可回答與跨來源驗收題 - 提出更新與權限複查清單 禁止自動執行： - 自動上傳、刪除或分享文件 - 把未核准個資、合約或財務資料列為可上傳 - 找不到來源時以一般知識補寫專案事實 必須人工批准： - 每份來源的納入與權限 - 分享對象與可回答範圍 - 過期來源的封存或移除 停止並轉人工處理： - 來源日期或版本不明時列為待確認 - 兩份來源內容衝突時並列差異並停止定論 - 權限不足或敏感度不明時排除於匯入清單外

建議格式

應該得到什麼

  • 來源納入清單
  • 命名與版本規則
  • 驗收問題
  • 更新維護清單

人工負責

交付前驗收

  • □ 知識庫範圍是否清楚
  • □ 每份來源是否有日期、版本與權限
  • □ 測試是否包含答不出來的問題
  • □ 分享設定是否經資料負責人確認
