三步上手

怎麼使用這個流程

  1. 列出 Agent 會讀取、產生、修改與傳送的內容
  2. 依影響與可逆性設定人工批准點
  3. 設計錯誤偵測、停止、回復與稽核紀錄

AI Agent 專用

執行設定與治理規格

運作模式：人工啟動的上線前與定期風險審查；只分析流程設定，不取得或測試真實權限。

觸發條件

  • Agent 首次設計完成準備試辦
  • 資料來源、模型、工具、權限或輸出對象變更
  • 發生錯誤、申訴、外洩疑慮或定期複查日期到期

允許的資料來源

  • 最新流程圖、權限矩陣與資料流向
  • 外部動作、責任人與回復機制
  • 組織個資、資安、發布、採購與專業審查規範

狀態流程

  1. 待審查
  2. 資產與資料盤點
  3. 逐步風險評估
  4. 人工關卡設計
  5. 停止與回復測試
  6. 等待風險負責人批准
  7. 准予試辦、限期改善或禁止上線

允許執行

  • 逐步評估資料、權限、可逆性與錯誤後果
  • 設定人工批准、停止條件與最小權限
  • 產生失敗情境、回復演練與稽核清單
  • 指出責任人缺漏或控制措施不足

禁止自動執行

  • 索取密碼、API 金鑰或真實個資
  • 自動核准 Agent 上線或擴權
  • 為通過審查而降低或省略風險等級

必須人工批准

  • 所有對外發布、寄送、金錢、資格、個資與不可逆動作
  • 權限配置、正式試辦與上線
  • 事故後重新啟用與控制措施例外

例外與失敗處理

  • 責任人不明或無法回復時判定不得上線
  • 未通過停止或回復測試時退回改善
  • 發現重大資料或權益風險時立即升級給風險負責人

最低紀錄要求

  • 流程版本、模型、工具、資料來源與權限範圍
  • 每項風險、控制措施、負責人與批准紀錄
  • 測試結果、事故、人工介入與回復時間

成效指標

  • 高風險動作人工批准覆蓋率 100%
  • 停止與回復測試通過率 100%
  • 重大事故、越權或未授權外部動作件數為 0
  • 定期複查與改善措施準時完成率
