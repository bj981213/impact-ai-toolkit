# 公益影響力 AI 工作箱｜60 分鐘講師流程

## 0–10 分鐘：先分清 Prompt 與 Agent

- Prompt 適合單次任務；Agent 適合需要讀取工具、依情境做有限判斷並重複執行的任務。只有固定規則與欄位搬運時，優先使用一般自動化。
- AI 負責整理、草擬與檢查；人負責資料來源、個資、權限與最後核准。

## 10–25 分鐘：日常工作

- 從「會議紀錄與待辦整理」說明：原始紀錄、決議、待辦、負責人與期限要分開。
- 搭配「文件重點摘要」與「文件版本差異比較」，示範如何要求 AI 保留頁碼、段落或章節位置。
- 用「每週／每月專案進度摘要」把完成事項、阻礙、風險與下期工作整理成主管能快速判斷的格式。

## 25–40 分鐘：專案、提案與報告

- 用「活動企畫與執行表」整理目標、對象、流程、分工、時程與風險。
- 用「問卷題目設計」檢查一題一意、選項完整、填答負擔與個資必要性。
- 用「補助／獎項申請檢查」逐題核對是否漏答、超字數或缺少附件與證據。
- 報告可從「一頁主管摘要」或「成果報告架構與初稿」開始，沒有資料的地方只能標示缺口，不能補寫。

## 40–50 分鐘：事實查核與個資安全

- 對外發布前用「對外內容事實查核」逐項核對人名、職稱、日期、數字、引述與連結。
- 資料貼入 AI 前用「AI 使用前個資安全檢查」判斷哪些欄位要刪除、遮蔽、概括化或改用虛構資料。
- 查不到來源時標示「未核實」；涉及個案身分、健康、財務或聯絡方式時停止上傳並轉人工處理。

## 50–57 分鐘：Agent 指令怎麼用

- 從會議紀錄、專案進度、信件與表單待辦、文件整理與公開資訊追蹤五種重複工作中選一種。
- 選擇一個工具，替換所有以 `[ ]` 標示的欄位，再複製完整指令。
- 將指令貼入使用中的 AI Agent；第一次先用虛構資料測試。
- 檢查輸出是否符合頁面上的驗收清單，再決定是否採用。

## 57–60 分鐘：帶走網站與 QR Code

- 只做一次的工作從「Prompt 工具箱」開始；需要固定週期、固定資料與人工關卡的工作，改看「AI Agent 工具箱」。
- Prompt 與 Agent 頁面都提供可直接複製的完整指令。
- 從工作需求地圖找任務，而不是從 AI 品牌找工具。
- 說明搜尋、收藏、一鍵複製與最近看過；不安排現場示範流程。
- 正式網址為 `https://bj981213.github.io/impact-ai-toolkit/?v=2.8.0`；請以此網址產生 QR Code，投影片同時保留文字網址，避免掃碼失敗或讀到舊版快取。
- 提醒觀眾：網站不接收檔案，也不會替他們把內容送進任何 AI。
